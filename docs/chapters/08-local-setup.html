ローカル環境構築

Linuxマシン上でOpenClawを構築する手順を解説します。 ここでは例として、小型Linuxマシン（Ubuntu 24.04 LTS）を使用します。

構築の流れ

前提条件

項目 説明
OS Ubuntu 24.04 LTS（CUI環境）
Docker v29.x 以上
APIキー OpenClawで使用するLLMのAPIキー

手順1: Ubuntu初期セットアップ

# スリープ無効化（常時稼働の場合）
sudo systemctl mask sleep.target suspend.target hibernate.target hybrid-sleep.target

# Docker公式リポジトリからインストール
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# ログアウト → 再ログイン（dockerグループ反映用）

手順2: ディレクトリとファイルの準備

mkdir -p ~/openclaw-dev ~/.openclaw

テンプレートから設定ファイルを作成

# テンプレートをコピー
cp .env.template ~/openclaw-dev/.env
cp docker-compose.yml.template ~/openclaw-dev/docker-compose.yml
cp openclaw.json.template ~/.openclaw/openclaw.json

各ファイルに実際の値を設定

  • .env: APIキーを設定
  • openclaw.json: Discord等の設定値を実際の値に置換

手順3: Dockerイメージをビルド

cd ~/openclaw-dev
docker build -t openclaw-local .
  • 所要時間: 5〜15分
  • ベース: 公式イメージ
  • 追加パッケージ: Chromium、ffmpeg、git、jq、python3 等

手順4: コンテナを起動

cd ~/openclaw-dev
docker compose up -d

手順5: 動作確認

# コンテナ状態
docker compose ps
# → openclaw-gateway が Up (healthy) であること

# ログ確認
docker compose logs -f --tail 50

# Gateway応答
curl http://127.0.0.1:18789

手順6: ファイル権限設定

sudo chown -R 1000:1000 ~/.openclaw/
sudo chmod -R 775 ~/.openclaw/

手順7: VPN設定（外出先からアクセスする場合）

# Tailscale VPN（外出先から安全にアクセス）
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

TailscaleのIPアドレスで外出先からSSH接続可能です。

アップデート手順

cd ~/openclaw-dev

# 1. ベースイメージを最新化
docker pull ghcr.io/openclaw/openclaw:latest

# 2. カスタムイメージを再ビルド
docker build -t openclaw-local .

# 3. コンテナ再作成
docker compose up -d

# 4. 動作確認
docker compose ps
docker compose logs --tail 20

注意: docker compose pull ではなく docker pull + docker build を使ってください。 カスタムDockerfileでビルドしているため、pull だけでは反映されません。

設定変更時の手順

openclaw.json 編集後

# 構文チェック
python3 -m json.tool ~/.openclaw/openclaw.json > /dev/null && echo OK || echo ERROR

# 再起動（restart ではなく stop && up）
cd ~/openclaw-dev && docker compose stop openclaw-gateway && docker compose up -d openclaw-gateway

.env 編集後

cd ~/openclaw-dev && docker compose stop openclaw-gateway && docker compose up -d openclaw-gateway

トラブルシューティング

コンテナが起動しない

docker compose logs gateway

ログからエラー原因を特定してください。

権限エラー（ファイルが編集できない）

sudo chown -R 1000:1000 ~/.openclaw/
